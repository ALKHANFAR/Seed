# Core OS Philosophy — Fundraising as an Operating System

> Adapted from the builder philosophy behind gstack/gbrain (Garry Tan) and Zolidar's
> context-first pitch building, redirected entirely at one job: closing investors.

## The Translation

| Builder world (gstack) | Fundraising world (Seed OS) |
|---|---|
| Think → Plan → Build → Review → Test → Ship → Reflect | Truth → Thesis → Materials → Review → Pressure-test → Send → Retro |
| CEO Review challenges scope | Deal CEO Review challenges the investment thesis |
| Eng Review locks architecture | Diligence Review locks the proof architecture |
| QA opens a real browser | Investor QA attacks the deck like a skeptical VC |
| /ship opens the PR | /investor-package sends a verified package |
| /retro improves the team weekly | /fundraising-retro improves the round weekly |
| gbrain remembers everything | Investor Brain remembers every meeting, question, objection |

## Five Laws of this OS

1. **Truth before slides.** The deck is the last surface. The system underneath
   (context, evidence, memory) is the product. (Zolidar's core lesson.)
2. **No claim without a gate.** Every investor-facing statement passes a proof gate
   and carries a tag: VERIFIED / PARTIAL / MISSING.
3. **Memory compounds.** Every meeting, question and objection is captured in the
   Brain. The 10th investor meeting must be 10x sharper than the 1st. (gbrain's lesson.)
4. **Agents, not heroes.** Reviews are run by named agent roles with checklists,
   not by founder mood. (gstack's lesson.)
5. **Weekly loop or death.** Fundraising without a retro loop decays. Every week:
   what moved, what stalled, what we learned, what we change.

## What investors actually buy

Investors do not buy Slidev, gbrain, or gstack. They buy:
problem size, distribution, moat, proof, and a path to venture-scale ARR.
This OS exists to make those five things undeniable and reusable.
