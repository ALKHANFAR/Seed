# The Operating Loop

## Daily (15 min)
1. Open `crm/pipeline.md` — what is the next action for every live investor? Do it or schedule it.
2. Capture: any new meeting/question/objection from yesterday → `brain/` pages.
3. One proof upgrade: move one claim from MISSING → PARTIAL or PARTIAL → VERIFIED.

## Per investor meeting
```
BEFORE  → run /investor-office-hours (prep brief from Brain)
DURING  → capture verbatim questions + objections
AFTER   → within 2 hours: update brain/ + crm/ + objection tracker
        → run /investor-learn (what did this meeting teach the system?)
FOLLOW  → run /investor-package if materials promised
```

## Per material (deck, memo, model, answer)
```
DRAFT → /deal-ceo-review → /deck-review or /financial-review or /diligence-review
      → /investor-qa (skeptical pass) → proof-gates check → SEND
```
Nothing reaches an investor without at least: one CEO review + one QA pass + gate check.

## Weekly (45 min, non-negotiable)
Run `/fundraising-retro`. Update `readiness/term-sheet-readiness-score.md`.
If the score did not move in 2 weeks, the strategy is wrong — escalate to thesis review.
