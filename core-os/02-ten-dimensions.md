# 10 Dimensions Framework — Scoring Rubric

Every major output (deck, memo, narrative, data room) is scored 0–10 per dimension.
A Seed-ready package needs: **no dimension below 6, average above 7.5.**

| # | Dimension | 10 looks like | Common failure |
|---|---|---|---|
| 1 | Technical | Live product demo + architecture doc + uptime/usage data | "AI-powered" with no demo |
| 2 | Memory | Usage compounds value; data network effect is measurable | Data collected but unused |
| 3 | Story | One-liner repeatable by the investor to their partners | Feature list, no narrative |
| 4 | Team | Founder-market fit obvious in 2 sentences + execution velocity proof | Generic bios |
| 5 | Product | Magic moment shown in <60s; clear time-to-value | Tour of menus |
| 6 | Operations | Revenue/employee, ship cadence, agent-run reviews visible | "We work hard" |
| 7 | Moat | Compounding data + switching cost + regulatory fit named with evidence | "First mover" |
| 8 | Scale | Bottom-up SOM → GCC path with named channels | "1% of $X B market" |
| 9 | Impact | Vision-2030 / localization alignment with a concrete tie | Vague ESG language |
| 10 | Return | Comps + ownership math + exit logic to ≥10x | No return story at all |

## How to score
For each dimension write: score, one line of evidence (with tag), one upgrade action.
Store completed scorecards in `retros/` with the date.
