# Proof Discipline — Claim → Proof → Slide → Answer → Asset

The single pipeline every piece of information travels through:

```
RAW INFO (note, metric, contract, quote)
   ↓ classify
CLAIM        e.g. "Sondos reduces no-shows materially"
   ↓ proof-gates/
TAG          VERIFIED (doc/metric exists) / PARTIAL / MISSING
   ↓ if VERIFIED or PARTIAL-with-caveat
SLIDE        an Action-Title slide candidate in pitch-factory/
   ↓
ANSWER       a 3-sentence investor answer in brain/ (Q&A bank)
   ↓
ASSET        the underlying document filed in data-room/ with an index entry
```

## Rules
- A claim with tag MISSING may exist internally but **never** in investor materials.
- PARTIAL claims must be phrased as hypotheses or roadmap ("early data suggests…").
- Every VERIFIED claim must name its asset: `[proof: data-room/03-traction/no-show-study.pdf]`.
- Re-verify all VERIFIED claims monthly — proofs go stale.
