# Confusion Protocol, the 3-Strikes Rule & Checkpoint Discipline

> Three meta-protocols from gstack that govern HOW every other command behaves.

---

## 1. Confusion Protocol

For **high-stakes ambiguity** — round structure, valuation strategy, whether to
take strategic money, conflicting partner signals, anything resembling a one-way
door — the system must STOP. It then:

1. **Names the confusion in one sentence.** ("It's unclear whether STV's silence
   means the partner passed internally or is waiting on the metrics pack.")
2. **Presents 2-3 options with tradeoffs.** Not one recommendation dressed as a question.
3. **Asks. Then waits.**

Do NOT use this for routine work (formatting, follow-up wording, CRM updates) —
that's hiding behind questions. The protocol exists for genuine forks, not for
outsourcing courage.

---

## 2. The 3-Strikes Rule (from /investigate)

In code: if three fix attempts fail, stop patching and question the architecture.
In fundraising:

**If three different rebuttals to the same objection fail across meetings —
stop rewording. The problem is not the wording. It is the narrative architecture.**

Examples:
- Three funds independently choke on "two products, one company" → you don't need
  a better slide; you need a structural answer (holding-co story, sunset plan, or
  a sequencing argument).
- Three partners ask "why now?" and stay unconvinced → the timing thesis is weak,
  not the phrasing. Go back to `/investor-office-hours` and rebuild it.
- Three meetings end with "come back with more traction" → the round is early.
  That's a strategy fact, not a pitch bug.

When the rule triggers, the system escalates from *tactical* commands
(`/feedback-triage`, objection ladder) to *strategic* ones (`/investor-office-hours`,
narrative rebuild) — and logs the escalation in the brain so it's never relearned.

## 3. Continuous Checkpoint Mode

In code: auto-commit WIP with a structured context block, so a crash never loses
the session. In fundraising: **every investor touch is committed to the brain the
same day, with a structured context block:**

```
TOUCH: <fund> · <person> · <channel> · <date>

[seed-context]
Decisions: <what we chose to say / show / concede>
Signals:   <what they reacted to — positive and negative, verbatim where possible>
Remaining: <what we owe them, what they owe us, deadline>
Tried:     <rebuttals or framings that did NOT land — worth recording>
[/seed-context]
```

Rules:
- Log only real signal, never `git add -A` (don't dump the whole meeting transcript
  as "notes" — extract decisions, signals, remaining, tried).
- Never log mid-negotiation speculation as fact.
- `/context-restore` reads these blocks to brief any session (or any co-founder)
  in 60 seconds. `/investor-package` squashes them into the clean fund history.

**Why it matters:** fundraising sessions are long-running and interrupted constantly.
The founder who can reconstruct exact deal state after two weeks of product firefighting
beats the founder who "remembers roughly where things stood."

---

## 4. Context Health (soft directive)

During any long working session, periodically write a `[PROGRESS]` line:
done / next / surprises. If the session is looping — same objection, same slide,
same rewrite for the third time — STOP and reassess. Looping is the signal to
either invoke the 3-Strikes Rule or save context and walk away. Progress notes
never change any outbound material.
