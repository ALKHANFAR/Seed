# حالة العرض الاستثماري — عشر أبعاد القابضة (DECK BUILD STATE)

> **الغرض من هذا الملف:** سجل الحقيقة الوحيد (Single Source of Truth) لبناء العرض الاستثماري شريحة‑بشريحة.
> أي LLM يقرأ هذا الملف يجب أن يكون قادراً على الانطلاق فوراً من الشريحة التالية غير المعتمدة، بدون إعادة أي قرار سابق.
> آخر تحديث: 13 يونيو 2026.

---

## 0. القواعد التشغيلية (اقرأها أولاً)

1. **نمشي شريحة‑بشريحة.** لكل شريحة: (أ) نعتمد المحتوى نصاً، (ب) نصمّمها، (ج) المستخدم يعدّل أو يعتمد، (د) ننتقل.
2. **لا ننتقل لشريحة جديدة قبل كلمة "اعتمد" صريحة من المستخدم على الشريحة الحالية.**
3. **أسلوب المحتوى:** كل كلمة محسوبة وعميقة. لا حشو، لا تكرار. نستبدل الجمل الوصفية برموز/أسهم (مثل `مكالمة ← قرار`) حيثما أمكن. التشذيب ذكي لا عشوائي.
4. **التصميم:** المستخدم يحب التصميم الحالي للعرض — نحافظ عليه بالكامل. التعديل على المحتوى والأبعاد فقط، لا على الهوية البصرية.
5. **الأبعاد:** كل شريحة مبنية داخلياً على لوحة `.ppt-slide` بحجم **1280×720** (نسبة 16:9). المستخدم طلب ترقية العرض النهائي إلى **1920×1080** — يُنفّذ عبر `transform: scale(1.5)` على الإطار الخارجي (`.slide-wrap`)، **بدون لمس التصميم الداخلي للشرائح إطلاقاً** (تغيير الأبعاد الداخلية يكسر التخطيط).
6. **اللغة:** عربي RTL أساسي، مع سطر إنجليزي مختصر تحت العناوين (قرار معلّق — انظر القسم 4).

### المنتجان (التعريفات المعتمدة — مصدرها Mem، تحقّقنا منها):
- **سندس (Sondos):** `مكالمة ← قرار`
  - الجوهر (من Mem · وثيقة Sondos Reset): نظام ذكاء صوتي/محادثاتي (Voice AI) يحوّل المكالمة/الطلب إلى **قرار وتنفيذ في ثوانٍ** (الميزة المميِّزة: قرار في 12 ثانية بدل 48 ساعة). حيّ في الإنتاج · 11 عميل يدفع · 58K MRR.
- **سيادة (Siyadah):** `رابط ← موظفين رقميين`
  - الجوهر (من Mem · استراتيجية URL→Everything + التحكم البرمجي الكامل): العميل يضع رابط موقعه، فيبني النظام **موظفين رقميين** (Digital Employees / 6 وكلاء) يربطون أنظمته تلقائياً. ليست "أتمتة" — بل "موظفون رقميون". في Beta.
  - **مهم:** المستخدم رفض صراحةً كلمة "أتمتة" — دائماً نقول "موظفين رقميين".

### قرارات صياغة مرفوضة/بديلة (لا تُعاد):
- سندس: رُفض "الموظف الصوتي للقطاع الخدمي" و"موظف صوتي يُنفّذ". المعتمد = `مكالمة ← قرار`.
- سيادة: رُفض "نظام التشغيل الذاتي للمنشأة" وأي صياغة فيها "أتمتة". المعتمد = `رابط ← موظفين رقميين`.

## 2. الأصل البصري: الشعار (معتمد نهائياً)

الشعار = قطرة/شعاع (teardrop) — **يُرسم كـSVG vector نقي** (لا صورة JPEG/PNG، لأن الراستر يطلع مبكسل). الشكل الصحيح: حافة علوية أفقية مستقيمة + دائرة كاملة يميناً + قاع أيسر مائل **ينحني بنعومة** (Bézier) للاتصال بالدائرة بدون زاوية حادة.

**كود SVG النهائي المعتمد (مطابق للأصل الذي أرسله المستخدم):**

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 540 296" fill="none">
<path d="M0 1 H391 A147 147 0 1 1 300 286 Q180 230 0 1 Z" fill="currentColor"/>
</svg>
```

- في **الوسط (hero):** `class="hero-drop"` بعرض ~240px، `color:#fff`، مع توهج:
  `filter: drop-shadow(0 0 60px rgba(59,130,246,0.45)) drop-shadow(0 0 120px rgba(99,102,241,0.25));`
  **مهم:** أُزيل فلتر `brightness(0) invert(1)` القديم لأنه يقتل لون الـSVG.
- في **الشريط العلوي (topbar):** `class="h-7 w-auto text-white"` + `filter: drop-shadow(0 0 6px rgba(96,165,250,0.4))`.
- استبدلنا كل صور الـbase64 القديمة بهذا الـSVG في كل المواضع.

## 5. الملفات والبيئة التقنية

- **الملف المصدر الأصلي:** `عرض_استثماري_عشر_ابعاد_القابضة_.html` (13 شريحة، كل واحدة داخل `<iframe srcdoc="...">` في الملف الأب). البنية: شريط علوي + stage + شريط سفلي تنقّل + JS تنقّل بالأسهم/الأرقام.
- **استخراج شريحة:** `re.search(r'<iframe class="slide-frame[^"]*" data-idx="N" srcdoc="(.*?)"></iframe>')` ثم `html.unescape()`.
- **المعاينة:** Tailwind v3 لا يُحمّل من CDN في بيئة الساندبوكس (محجوب). الحل: بناء CSS محلياً عبر `npx tailwindcss -i tw-input.css -o tw-built.css` ثم حقنه بدل سطر الـCDN، ولقطة عبر Playwright (`device_scale_factor=2`, لقطة عنصر `.ppt-slide`).
- **خطوط:** Cairo + Tajawal من Google Fonts.
- **ألوان الهوية:** أزرق `#60a5fa→#3b82f6` (سندس)، بنفسجي `#a78bfa→#8b5cf6` (سيادة)، كهرماني `#fbbf24→#d97706` (تحذير/أرقام صادمة)، أخضر `#34d399` (إيجابي/نمو)، خلفية سوداء متدرّجة.
